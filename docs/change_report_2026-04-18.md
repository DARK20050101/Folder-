# DiskExplorer 修改报告（第二轮）

日期：2026-04-18

## 本次需求覆盖

1. “返回上一级对比”仅在点击“与缓存对比”进入对比模式后出现。
2. 新增独立“重新扫描”按钮，重扫结果自动保存为新快照（沿用盘路径+时间格式）。
3. 优化盘符右键：移除无意义展示，新增“选择该盘历史缓存”。
4. 左键点击盘符时：
	- 有历史缓存：弹窗选择快照（默认最新）并显示当前使用快照。
	- 无历史缓存：自动触发重扫。
5. 视图菜单改为“空间历史变化”：
	- 先选磁盘；
	- 生成一级目录历史变化折线图；
	- 点击一级目录进入二级目录历史变化；
	- 点击返回按钮回到上一级。
6. 代码稳定性与性能优化，并形成复查报告。

## 主要改动

### 1. 盘符交互重构

- 盘符按钮左键：改为优先弹窗选择历史缓存快照。
- 盘符按钮右键：新增“选择该盘历史缓存”“重新扫描”。
- 工具栏标题字符串移除（不再显示无意义的 disks 标识）。

### 2. 重新扫描自动保存

- 新增主界面“重新扫描”按钮。
- 从重扫入口触发的扫描，完成后自动落盘缓存。
- 普通扫描仍保留“退出前确认保存缓存”机制，兼顾手动控制。

### 3. 双状态显示（当前快照 + 对比基线）

- 状态栏新增“当前快照”标签：显示当前页面使用的是哪次历史缓存。
- 原“历史缓存基线”标签保留，用于对比模式下的基线快照显示。

### 4. 对比模式与返回按钮可见性

- “返回上一级对比”按钮仅在对比模式中显示。
- 非对比模式（普通浏览、空间历史变化）自动隐藏。

### 5. 空间历史变化（折线图）

- 新增历史趋势图组件 `HistoryTrendChartWidget`。
- 数据源为同盘多快照（按时间序列）。
- 默认展示一级目录趋势；点击一级目录进入对应二级目录趋势。
- 新增历史模式返回按钮，回到一级目录趋势。

## 稳定性与效率优化

- 状态机优化：统一对比模式、历史模式、普通模式切换，避免状态残留。
- 快照加载防御：加载失败快照自动跳过，不中断历史图生成。
- 历史图快照上限：最多加载最近 40 份快照，降低大缓存目录下的绘图开销。
- 统一 UI 刷新：封装当前快照/基线标签刷新与按钮可见性刷新函数，减少重复逻辑。

## 复查建议

- 若后续快照量进一步增大，可增加异步历史数据聚合缓存（减少重复 `find`）。
- 若需要更强审计能力，可在快照元数据增加“触发来源”（左键加载/右键重扫/刷新等）。

## 第三轮修复（本次）

- 历史缓存名称展示增强：
	- 状态栏持续显示“当前快照：{名称}”；
	- 路径栏在加载历史缓存时追加当前快照名称。
- “历史空间变化”重构为独立窗口：
	- 新增独立窗口，风格与主界面保持一致；
	- 增加“返回”和“退出对比”按钮；
	- 通过滚动区域承载图表，解决无法上下滑动问题。
- 卡顿优化：
	- 历史窗口对每层目录结果做缓存；
	- 行数据构建改为按快照先生成子目录大小映射，减少重复树查找；
	- 历史图渲染快照上限提高到 60 份时仍保持可用性。

## 第四轮修复（稳定性与速度）

- 闪退防护（高内存压力）：
	- 扫描线程增加 `MemoryError` 专门捕获并给出可读错误提示；
	- 缓存反序列化遇到 `MemoryError` 时不崩溃、不误删缓存文件，改为安全返回失败。
- 加载缓存闪退防护：
	- 盘符左键默认自动选择“最新历史缓存”；
	- 若最新缓存加载失败（低内存/损坏），自动回退到重扫并自动保存。
- 性能优化（对比缓存与历史缓存对比）：
	- 缓存保存时写入 `meta` 侧文件，枚举历史快照不再反序列化整棵树；
	- 缓存保存时写入 `trend` 侧文件，历史趋势优先读取轻量摘要，显著降低内存占用和加载延迟；
	- 历史趋势数据加载统一改走轻量 API，减少大量 `FileNode.find` 与对象驻留。

## 第五轮修复（视图与历史缓存管理）

- 修正“刷新 / 重新扫描”文案，去掉可能导致显示异常的组合文本。
- 新增“删除当前缓存”功能，可直接删除当前正在使用的快照，并同步刷新状态栏和对比状态。
- “全部历史缓存”改为优先读取 `meta` 侧文件并缓存索引，减少菜单打开时的即时 I/O 压力。
- 删除缓存后会立即刷新历史缓存菜单索引，避免界面残留过期条目。
